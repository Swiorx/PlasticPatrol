# PlasticPatrol — Hackathon Pitch Package

**Pitch slot:** 2026-04-26, 15:00 EEST
**Format:** 3 min pitch + 2 min Q&A
**Challenge:** Protecting Water Resources from Pollution
**Judging:** Relevance 33% · Innovativeness 33% · Team 33%

## What's in this folder

| File | Purpose |
|---|---|
| `PITCH_SCRIPT.md` | Word-for-word 3-min speaker script, timing-marked per slide |
| `SLIDE_DECK_OUTLINE.md` | 9 slides — content, visuals, layout — ready to paste into Google Slides |
| `DEMO_SCREENCAST_SHOTLIST.md` | 30-second screencast shot list with on-screen captions |
| `BUSINESS_PLAN.md` | 4-page business plan (hybrid B2G + B2B + B2C with sponsors and 3-yr projection) |
| `QA_PREP.md` | 8 anticipated questions with prepared 30-second answers |

## Reading order for the team

1. **`PITCH_SCRIPT.md`** — read this first to internalise the narrative arc
2. **`SLIDE_DECK_OUTLINE.md`** — build the deck from this; one slide per section
3. **`DEMO_SCREENCAST_SHOTLIST.md`** — record the 30s demo per this; embed in slide 4
4. **`BUSINESS_PLAN.md`** — submit alongside the deck; export as PDF
5. **`QA_PREP.md`** — rehearse out loud before the pitch

## Pitch-day checklist

- [ ] Slides built from `SLIDE_DECK_OUTLINE.md` in Google Slides
- [ ] Screencast recorded per `DEMO_SCREENCAST_SHOTLIST.md`, embedded in slide 4 (NOT linked — file embedded so it autoplays without internet)
- [ ] Pitch script rehearsed to 175–180 seconds (3 full run-throughs minimum)
- [ ] Business plan exported as PDF and submitted with deck
- [ ] Q&A sheet printed for the speaker; 2-min Q&A budget rehearsed
- [ ] Backup speaker briefed on script in case of last-minute swap
- [ ] Laptop tested with venue projector (autoplay enabled, presenter view working)
- [ ] Repo QR code generated and added to closing slide

## Hard constraints

- **3-minute hard cap.** Going over = penalised or cut off.
- **Demo MUST be embedded in the submitted slides** — pre-recorded screencast required.
- **Live demos are forbidden** during pitch and Q&A.
- **2-min Q&A** — answers should be ≤ 30 seconds each.
